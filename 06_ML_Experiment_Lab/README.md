# ML Experiment Lab

## Product Name

ML Experiment Lab.

## Problem

ML notebooks often show code but not product relevance. AI Tech Quest needs a place to keep experiments, baselines, evaluation notes, and model artifacts without turning every experiment into a standalone product.

## Target Users

- Qingyu, as the technical researcher and builder.
- Interviewers reviewing ML fundamentals.
- Engineers checking experiment discipline.
- Future product modules that need model evidence.

## Core Features

- Colab experiment notes.
- Baseline model comparisons.
- Feature engineering notes.
- Accuracy and confusion matrix reports.
- Error analysis.
- Dataset and model version notes.
- Exportable artifacts for AI Tech Quest demos.

## Tech Stack

- Python.
- Google Colab.
- scikit-learn.
- pandas.
- NumPy.
- Matplotlib or Plotly.
- Markdown reports.

## MVP Scope

ML Experiment Lab is research support, not a main product.

MVP scope:

- Keep one clean classification experiment.
- Export metrics and example predictions.
- Use results to support AI Tech Quest Level 2.
- Keep notebooks readable and reproducible.

## Monetization

This module is not the primary monetization product.

It supports monetization indirectly by:

- Strengthening AI Tech Quest technical credibility.
- Supporting client demos that need model evaluation.
- Providing evidence for ML-related interviews.

## Roadmap

1. Choose one classification dataset.
2. Build baseline model notebook.
3. Add evaluation report.
4. Export metrics for frontend display.
5. Connect model-serving endpoint later if needed.
6. Keep this module as supporting evidence, not the main sales offer.
