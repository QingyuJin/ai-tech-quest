# Portfolio Website

## Product Name

Portfolio Website.

## Problem

AI Tech Quest is an interactive showcase, but Qingyu also needs a clear public website that can present identity, services, case studies, contact paths, and links to product demos without forcing every visitor through the full game flow.

## Target Users

- Recruiters.
- Interviewers.
- Freelance clients.
- Collaborators.
- Visitors who want a quick overview before exploring AI Tech Quest.

## Core Features

- Landing page for Qingyu's AI and web product work.
- Case study index.
- Contact / hire section.
- Links to AI Tech Quest and product demos.
- Links to GitHub repositories.
- Screenshots and walkthrough videos.
- Future blog or technical notes.

## Tech Stack

- React or Next.js.
- Tailwind CSS.
- Static deployment on Vercel, Netlify, or GitHub Pages.
- Markdown case studies.
- Future CMS or lightweight content system if needed.

## MVP Scope

- Create a clean portfolio home page.
- Link to AI Tech Quest as the main interactive product.
- Add cards for AI Business Assistant, TW Civic RAG, BuildFlow, Unity AI Tutor, and ML Experiment Lab.
- Add contact CTA.
- Add placeholder case study links.

## Monetization

The Portfolio Website is an acquisition and trust layer.

It supports:

- Interview conversion.
- Freelance inquiry conversion.
- Product demo discovery.
- Clear presentation of service packages.

## Roadmap

1. Define portfolio copy and service positioning.
2. Add project cards and screenshots.
3. Add case studies.
4. Add contact form or direct contact links.
5. Deploy publicly.
6. Connect analytics after public release.
